# Search 业务逻辑说明

本文档介绍 Paper Tracker 执行搜索时的核心业务逻辑，包括分页拉取、早停、去重、多数据源聚合、时间窗口过滤等机制。不涉及具体代码实现，只描述行为和规则。

---

## 整体流程

一次搜索按以下顺序执行：

1. 对配置文件中的每个 query，依次向所有启用的数据源（如 arXiv、OpenAlex）发起搜索
2. 各数据源独立执行分页拉取，每页结果在本源内去重后累积
3. 所有源的结果合并后，执行跨源去重，得到最终论文列表
4. 若启用 LLM 增强，批量生成翻译和摘要并注入到论文数据中
5. 将结果输出到配置的格式（控制台、JSON、Markdown、HTML）
6. **输出成功后**才执行持久化，将论文标记为"已见"并保存内容快照

持久化在输出之后执行，保证部分输出失败不会污染去重状态库。

---

## 分页拉取

数据源 API 每次只返回一批论文，系统通过多轮请求拼接完整结果。

### 分批逻辑

- 每次请求拉取固定数量的论文（`fetch_batch_size`）
- 累积拉取到满足条件为止（见下方停止条件）
- 每轮请求之间有速率限制等待，避免触发 API 限流

### 停止条件（满足任意一条即停止）

- 当前页为空，说明上游已没有更多结果
- 触发早停（见下节）
- 已收集到足够数量的新论文（达到 `max_results`）
- 已拉取论文总数达到上限（`max_fetch_items`，`-1` 表示无限制）
- 本次查询已耗时超过 120 秒

---

## 早停

早停是一种优化机制，利用 API 返回结果按时间倒序的特性，在确认后续页面不可能有新结果时提前终止分页。

**触发条件**：当前页中最旧的论文已在时间窗口之外。

由于结果是按更新时间倒序排列的，若一页里最旧的论文都已超出窗口，后续所有页面的论文时间更老，必然也在窗口外，因此可安全停止。

---

## 时间窗口

时间窗口决定了哪些论文在时间上满足收录条件。系统支持两种窗口模式：

### 严格窗口

`[当前时间 - pull_every 天, 当前时间]`

只收录最近 `pull_every` 天内更新过的论文。这是默认行为。

### 填充窗口（可选）

当 `fill_enabled = true` 时，同时开放一个更宽的回溯窗口：

`[当前时间 - max_lookback_days 天, 当前时间]`

- `max_lookback_days = -1` 表示无限回溯
- 用途：补充拉取历史期间漏掉的论文

**约束**：若启用填充且 `max_lookback_days` 不为 `-1`，则 `max_lookback_days` 必须 `≥ pull_every`。

### 论文纳入规则

同时满足以下任一条件的论文进入候选：

- 在严格窗口内
- 填充已启用，且在填充窗口内

## 去重

去重分三个层级，层层递进。

### 第一层：本源分页内去重

在分页拉取过程中，每拿到一页结果就立即过滤掉已在本次拉取中出现过的论文，避免同一篇论文在多页之间重复计数。

### 第二层：跨次去重（已见记录）

每次搜索结束并成功输出后，系统将本次新论文标记为 `seen` ，写入本地数据库。下次搜索时，已见论文不再计入新结果。

### 第三层：跨源批量去重

所有数据源拉取完毕后，对聚合结果做跨源去重，合并指向同一篇论文的不同记录。

**匹配依据（优先级从高到低）**：

1. **DOI**：规范化后完全相同（最可靠）
2. **指纹**：由规范化标题 + 第一作者 + 年份构成的字符串（保守匹配，标题需足够长）
3. **源 ID**：同一来源下的相同 ID

匹配到的多条记录合并为一条，按以下规则选出"赢家"：

- 优先选择类型为 `article` 且带有 DOI 的记录（最可信）
- 其次选择 `preprint` 类型的记录
- 同等条件下，按配置文件中的源顺序和更新时间决定

赢家保留主体数据，但若赢家缺少某些字段（如摘要链接、PDF 链接、作者列表），会从其他记录中回填。

## 多数据源聚合

系统支持同时从多个数据源搜索，当前支持 arXiv 和 OpenAlex。

### 并发与容错

- 各源依次执行，单源失败不影响其他源继续运行
- 仅当所有源全部失败时，本次查询才报错退出

### 结果排序

多源结果合并后按以下优先级排序：

1. 更新时间（倒序，最新在前）
2. 配置文件中的源顺序
3. 论文 ID

排序结果作为跨源去重的输入，排名靠前的论文在去重时优先作为候选赢家。

### 各源差异

| | arXiv | OpenAlex |
|---|---|---|
| 时间过滤 | 本地过滤 | API 参数 + 本地二次验证 |
| 分页方式 | 偏移量 | 页号 |
| NOT 过滤 | 编译进查询 | 本地后置过滤 |
| 类别字段 | 支持 | 不支持，跳过 |

---

## 查询作用域（scope）

配置文件支持定义一个全局 `scope`，它会应用到所有 query 上。

逻辑为：`(scope 条件) AND (每条 query 的条件)`

例如，设置 `scope.CATEGORY` 为 `cs.CV`，则所有 query 都只会返回该分类下的论文，无需在每条 query 里重复声明。

---

## LLM 增强（可选）

若配置了 LLM 服务，搜索结果在输出前会经过一次批量增强：

- 生成中文摘要翻译
- 生成结构化摘要（含动机、方法、结论等字段）

增强结果注入到论文数据的扩展字段中，仅影响本次输出内容，不改变原始论文数据。

---

## 相关配置项速查

| 配置项 | 含义 | 默认值 |
|---|---|---|
| `pull_every` | 严格时间窗口大小（天） | 7 |
| `fill_enabled` | 是否启用填充窗口 | true |
| `max_lookback_days` | 填充窗口最大回溯天数，`-1` 为无限 | 30 |
| `fetch_batch_size` | 每页拉取数量 | 25 |
| `max_fetch_items` | 单次查询最多拉取总数，`-1` 为无限 | 125 |
| `max_results` | 单次查询最多返回新论文数 | — |
